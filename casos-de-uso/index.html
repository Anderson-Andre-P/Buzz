1. Introdução

Os casos de uso são os veículos de captura de requisitos e servem de base para a definição de requisitos funcionais. Além de facilitarem a visualização da aplicação, ajudam na delimitação do sistema. Servem como meio de comunicação com usuários finais e clientes por oferecerem uma visão dinâmica e fechada do sistema. Outra característica dos casos de uso é a de servirem como base para a descoberta de objetos e servirem como ferramenta para a rastreabilidade de requisitos. Através deles pode-se definir a interface com o usuário, e ajudam na visualização de elementos desde requisitos funcionais até objetos e estruturas de componentes.


2. Diagramas de casos de uso

Parte da modelagem de casos de uso envolve a criação de um diagrama de casos de uso, que é uma forma de mostrar o modelo sem entrar nos detalhes dos casos de uso propriamente ditos. O sistema geralmente é delimitado em um diagrama de casos de uso por um retângulo com o nome do sistema identificado. O retângulo é chamado de limite do sistema. Os elementos que fazem parte do sistema estão dispostos dentro do limite. Entidades externas que não são parte do sistema e que interagem com este são mostradas do lado de fora. O nome do sistema é necessário para completar o primeiro passo. O nome geralmente é demonstrativo do seu objetivo, uma vez que os participantes devem poder facilmente identificar o que eles esperam do sistema e onde ele começa e termina.

O conjunto de casos de uso representa todas as possíveis interações que serão descritas nos requisitos de sistema. Um diagrama de caso de uso possui os seguintes elementos:

  • Casos de uso: podem ser descritos elementos que representam uma sequência de ações requeridas pelo sistema;
  • Atores: podem ser qualquer coisa fora do sistema que troque informações com o mesmo, incluindo usuários e outros sistemas;
  • Contêineres: um contêiner contém casos de uso de um sistema (limite do sistema). Pacotes5 podem ser utilizados para conter casos de uso e/ou atores e inclusive outros contêineres;
  • Relacionamentos: existem vários tipos de relacionamentos como, por exemplo, o de associação, que liga casos de uso e atores, indicando a forma como interagem; relacionamentos que representem dependência, por exemplo, entre dois pacotes; relacionamentos entre casos de uso, como o de “extend” e “include”; assim como a generalização, que pode, por exemplo, indicar a herança de comportamento entre dois atores;

3. Descrição dos casos de uso

3.1 Caso 1: Fazer cadastro

Caso 1 Informações
Descrição Quando um usuário acessar o aplicativo pela primeira vez depois de sua instalação, o ator necessitará fazer o cadastro no aplicativo para poder utilizar o transporte
Ator(es) Usuário
Pré-condições Ter o aplicativo instalado em seu smartphone e ter acesso à internet
Fluxo 1 - ator precisa baixar o aplicativo
2 - ator precisa abrir o aplicativo
3 - O ator precisa inserir seus dados no formulário de cadastro
4 - Se os dados inseridos forem válidos o usuário deverá concluir o cadastro
Pós-condições O ator poderá fazer login sempre que precisar e ter acesso aos seus dados dentro do aplicativo
Rastreabilidade -

3.2 Caso 2: Fazer login

Caso 2 Informações
Descrição O ator, ao iniciar o aplicativo, pode efetuar o login para utilizar o aplicativo
Ator(es) Usuário
Pré-condições Ter o aplicativo instalado, acesso a internet e já estar cadastrado no sistema
Fluxo 1 - o ator precisa abrir o aplicativo
2 - O usuário deve inserir seu email e senha para acessar a carteirinha virtual
Pós-condições O ator poderá utilizar o aplicativo
Rastreabilidade -

3.3 Caso 3: Ver carteirinha digital

Caso 3 Informações
Descrição Ao acessar a tela inicial do aplicativo, o ator poderá visualizar sua carteirinha digital e o QR Code de acesso ao transporte
Ator(es) Usuário
Pré-condições O ator deve estar logado no aplicativo
Fluxo 1 - ator precisa abir o aplicativo
2 - O ator precisa logar no sistema do aplicativo
3 - Pode visualizar sua carteirinha digital na primeira tela ou o QR Code na tela de QR Code
Pós-condições O ator poderá verificar todas as informações relacionadas a sua carteirinha digital
Rastreabilidade -

3.4 Caso 4: Ver se o transporte pode partir

Caso 4 Informações
Descrição Ao acessar a tela do Scanner de QR Code, o ator pode utilizar para ler o QR Code dos usuários do aplicativo que desejam entrar no transporte
Ator(es) Motorista
Pré-condições O ator deve estar logado no aplicativo e ter acesso a internet
Fluxo 1 - ator precisa abrir o aplicativo
2 - ator precisa realizar o login no aplicativo
3 - O ator clica no ícone do Scanner
Pós-condições O ator já pode escanear o QR Code da carteirinha digital dos usuários do aplicativo
Rastreabilidade -

3.5 Caso 5: Controlar quem pode usar o transporte

Caso 5 Informações
Descrição Ao acessar o aplicativo o ator pode utilizar o aplicativo para escanear o QR Code dos usuários do sistema para verificar se os mesmos podem ou não utilizar o transporte
Ator(es) Motorista
Pré-condições O ator deve estar logado e ter acesso a internet
Fluxo 1 - o ator precisa abrir o aplicativo
2 - o ator precisa logar no aplicativo
3 - na tela inicial o ator clica no ícone do Scanner
4 - o ator pode começar a escanear as carteirinhas digitais
Pós-condições O ator pode barrar a entrada de quem não tiver permissão para utilizar o transporte
Rastreabilidade -
